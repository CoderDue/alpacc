# Lisp Benchmarks

This folder contains lexer and parser benchmarks for the s-expression
grammar in `../../grammars/sexp.alp`. All implementations share a single
dataset: a grammatical s-expression generated by
`alpacc test generate --single-long`, sized by `INPUT_SIZE` (the number
of tokens; the payload is roughly twice that in bytes). The `.inputs`
file is framed for the CUDA CLI, and a raw copy with the 24-byte header
stripped is derived for the Futhark and standalone lexer benchmarks.

Run everything with:
```
$ make bench
```
or individually with `make bench-lexer` / `make bench-parser`, which in
turn run the per-implementation targets below. None of the benchmarks
account for the time to read the input into memory or the time to
generate the lexer/parser.

## Lexer benchmarks

### C lexer (`make bench-lexer-c`)
A handwritten lexer by Troels Henriksen (athas). It finds every token
in the input and does not save the tokens found.

### Rust lexer (`make bench-lexer-rust`)
Written in Rust using the lexer generator library Logos, which claims
to be fast and so seems like a good comparison. Like the Futhark lexer
it saves every token in memory, which costs some performance compared
to consuming tokens as you go.

### CUDA lexer (`make bench-lexer-cuda`)
An alpacc-generated CUDA lexer, benchmarked through its built-in CLI on
the framed `.inputs` file.

### Futhark lexer (`make bench-lexer-futhark`)
An alpacc-generated Futhark lexer, benchmarked with `futhark bench` on
the raw input.

## Parser benchmarks

### CUDA parser (`make bench-parser-cuda`)
An alpacc-generated CUDA combined lexer+parser, benchmarked through its
built-in CLI on the framed `.inputs` file.

### Futhark parser (`make bench-parser-futhark`)
An alpacc-generated Futhark combined lexer+parser, benchmarked with
`futhark bench` on the raw input.
